# Princípios do Projeto

## Objetivo

Este documento define os princípios permanentes que orientam as decisões técnicas, funcionais e organizacionais do projeto Caderno.

Os princípios possuem prioridade superior às implementações específicas e devem servir como referência para a evolução do software.

---

## 1. Conhecimento Acima de Conteúdo

O projeto não trata receitas apenas como textos.

Receitas, ingredientes, técnicas, equipamentos e observações são considerados conhecimento estruturado e devem ser modelados como entidades relacionadas.

---

## 2. Fonte Única da Verdade

Cada informação deve possuir uma localização oficial e claramente definida.

Duplicação de dados deve ser evitada sempre que possível.

Alterações devem ocorrer na origem da informação e propagar-se para os demais componentes.

---

## 3. Dados Primeiro

A prioridade do projeto é a preservação e organização dos dados.

Interfaces, ferramentas e funcionalidades devem existir para servir aos dados e não o contrário.

---

## 4. Simplicidade Antes da Complexidade

Soluções simples devem ser preferidas sempre que atenderem aos requisitos.

Complexidade só deve ser introduzida quando houver justificativa clara e documentada.

---

## 5. Evolução Incremental

O projeto deve evoluir por pequenas melhorias contínuas.

Grandes reescritas devem ser evitadas sempre que possível.

---

## 6. Código Aberto

O projeto deve permanecer aberto, auditável e acessível.

O conhecimento produzido deve ser preservado de forma transparente.

---

## 7. Independência Tecnológica

Sempre que possível, devem ser utilizados:

* Formatos abertos.
* Protocolos abertos.
* Ferramentas amplamente documentadas.
* Tecnologias maduras.

O projeto deve minimizar dependências desnecessárias.

---

## 8. Durabilidade

Os dados produzidos hoje devem continuar acessíveis no futuro.

A arquitetura deve priorizar:

* Portabilidade.
* Backup.
* Migração.
* Legibilidade.

---

## 9. Documentação Como Parte do Produto

Documentação não é um complemento.

Documentação faz parte do produto e deve evoluir junto com o software.

---

## 10. Qualidade Acima de Quantidade

Uma pequena coleção de informações corretas possui mais valor do que uma grande coleção de informações inconsistentes.

Todo conteúdo deve priorizar:

* Clareza.
* Precisão.
* Manutenção.
* Confiabilidade.

---

## 11. Preservação do Conhecimento

O objetivo final do projeto é preservar conhecimento culinário de forma organizada, pesquisável e sustentável ao longo do tempo.

Todas as decisões devem contribuir para esse objetivo.
